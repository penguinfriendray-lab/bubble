# 排序演算法視覺化報告

## 📋 報告概述

這份報告探討了常見的排序演算法，包括其原理、實作方式、時間與空間複雜度分析，以及互動式視覺化展示。

## 🎯 學習目標

- 理解不同排序演算法的基本原理
- 分析各演算法的時間與空間複雜度
- 透過視覺化動畫觀察演算法的執行過程
- 比較不同演算法的優缺點

## 📚 報告內容

### 1. 氣泡排序法 (Bubble Sort)
- **原理說明**: 重複比較相鄰元素並交換位置
- **時間複雜度**: O(n²)
- **空間複雜度**: O(1)
- **優點**: 實作簡單，穩定排序
- **缺點**: 效率較低，不適合大型資料

### 2. 選擇排序法 (Selection Sort)
- **原理說明**: 每次找出最小元素與當前位置交換
- **時間複雜度**: O(n²)
- **空間複雜度**: O(1)
- **優點**: 交換次數少，穩定排序
- **缺點**: 不穩定，效率較低

### 3. 插入排序法 (Insertion Sort)
- **原理說明**: 將元素插入已排序序列的適當位置
- **時間複雜度**: O(n²)
- **空間複雜度**: O(1)
- **優點**: 穩定排序，對小資料集效率高
- **缺點**: 大資料集效率低

### 4. 合併排序法 (Merge Sort)
- **原理說明**: 使用分治法將陣列分成兩半，遞歸排序後合併
- **時間複雜度**: O(n log n)
- **空間複雜度**: O(n)
- **優點**: 穩定排序，效率高，適合大型資料
- **缺點**: 需要額外空間，對於小資料集可能較慢

### 5. 快速排序法 (Quick Sort)
- **原理說明**: 選取基準值，將資料分成較小與較大兩部分，再遞迴排序
- **時間複雜度**: 最佳 O(n log n)，平均 O(n log n)，最壞 O(n²)
- **空間複雜度**: O(log n)
- **優點**: 平均效能佳，實務中常用
- **缺點**: 不穩定，最壞情況會退化，遞迴深度較高

### 6. 堆積排序法 (Heap Sort)
- **原理說明**: 先建立最大堆或最小堆，然後重複取出堆頂元素並重建堆
- **時間複雜度**: O(n log n)
- **空間複雜度**: O(1)
- **優點**: 最壞情況保證 O(n log n)，不需要額外大量空間
- **缺點**: 不穩定，實作較複雜，常數因子較高

## 🔍 綜合比較

### 排序演算法比較分析

#### 時間複雜度比較
- **O(n²) 演算法**: 氣泡排序、選擇排序、插入排序
  - 適用於小規模資料集 (n ≤ 1000)
  - 實作簡單，適合教學用途
- **O(n log n) 演算法**: 合併排序
  - 適用於大規模資料集
  - 效能穩定，適合實際應用

#### 空間複雜度比較
- **O(1) 演算法**: 氣泡排序、選擇排序、插入排序
  - 就地排序，不需要額外空間
  - 適合記憶體受限的環境
- **O(n) 演算法**: 合併排序
  - 需要額外的合併空間
  - 適合記憶體充足的環境

#### 穩定性比較
- **穩定排序**: 氣泡排序、插入排序、合併排序
  - 相同元素的相對位置保持不變
  - 適合需要保持原始順序的應用
- **不穩定排序**: 選擇排序
  - 可能改變相同元素的相對位置
  - 在某些情況下可能會有問題

#### 實務應用建議
- **小資料集 (n < 100)**: 插入排序 (最佳情況 O(n))
- **中等資料集 (100 ≤ n < 10000)**: 快速排序或合併排序
- **大型資料集**: 合併排序、堆積排序
- **需要穩定排序**: 氣泡排序、插入排序、合併排序
- **需要固定最壞時間**: 堆積排序
- **記憶體受限**: 氣泡排序、選擇排序、插入排序
- **教學用途**: 氣泡排序 (容易理解)
